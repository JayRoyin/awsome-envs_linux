# 示例运行

## G1 高层运行控制（远程开发，PC端开发）

获取[SDK](https://github.com/unitreerobotics/unitree_sdk2)
```bash
#下载SDK的zip或者git
git https://github.com/unitreerobotics/unitree_sdk2.git
#解压zip
unzip unitree_sdk.zip
```
首先进入工作空间`unitree_sdk`
```bash 
cd unitree_sdk
mkdir build

# 可以先确定Cmakelist.text
cd example
vim Cmakelist.text
# 注释相关不需要的
```
**vim用法**
>`i`进入插入模式
>
>`esc`退出插入模式
>
>`:wq`保存并退出Vim
>
```bash
# 进入build进行编译
cd ~/unitree_sdk/build/
cmake ..
make
# 如果make报错请注释相关的示例
```
进入相关的`bin`文件夹,并运行示例
```bash
cd bin
# 查看相关的可运行的示例
ls

# 查看192.168.123.164的网卡名称，例如enp2s0
ifconfig
# 运行示例
sudo ./g1_loco_client_example --network_interface=enp2s0 --set_velocity="0.5 0 0 1"
```


## G1 低层运行控制（本地开发，机器人端开发）

首先确认机器人本地是否有SDK，如果没有下载后传输过去
```bash
# 将进入SDK包所在的目录
scp unitree_sdk.zip unitree@192.168.123.164:~/
# 远程连接上机器人本地
ssh unitree@192.168.123.164
ls
# 解压
unzip unitree_sdk.zip
```
完成下载后进入`unitree_sdk`文件夹进行编译（与上述远程类似）

首先进入工作空间`unitree_sdk`
```bash 
cd unitree_sdk
mkdir build

# 可以先确定Cmakelist.text
cd example
vim Cmakelist.text
# 注释相关不需要的
```
**vim用法**
>`i`进入插入模式
>
>`esc`退出插入模式
>
>`:wq`保存并退出Vim
>
```bash
# 进入build进行编译
cd ~/unitree_sdk/build/
cmake ..
make
# 如果make报错请注释相关的示例
```
进入相关的`bin`文件夹,并运行示例
```bash
cd bin
# 查看相关的可运行的示例
ls

# 查看192.168.123.164的网卡名称，例如eth0
ifconfig
# 运行示例前，必须先吊起机器人，然后L2+Y进入零力矩，然后L2+R2,L2+A进入调试模式
sudo g1_ankle_swing_example eth0
```

## Go2示例开发
由于Go2和G1类似，就简单叙述
唯一的区别就是连接的ip不同，Go2的拓展坞的ip为`192.168.123.18`
### 高层运行控制
```bash
# enp2s0是PC端的网卡
sudo ./go2_stand_example enp2s0
```
如果用`go2_sport_client`，则修改`
const int TEST_MODE = stand_down`为相关动作后再编译后运行

### 低层运行控制
```bash
# eth0是机器人端的网卡
sudo ./go2_low_level eth0
```
